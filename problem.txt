Welcome to the Final Challenge of this Dev Weekend! 🛡️  In this challenge, you are auditing an internal HR Portal for a corporate web application. The provided server (app.py) features a standard file upload endpoint (/upload) designed to let employees upload new profile pictures. However, the backend file-saving logic is dangerously trusting.  This application contains a critical flaw known as Arbitrary File Write via Path Traversal. When a file is uploaded, the server blindly trusts the filename provided by the client and appends it directly to the local uploads/ folder path.  Modern web browsers automatically protect against this by stripping out malicious directory traversal characters (like ../). However, an attacker can bypass the browser entirely by writing an automated script to forge the raw HTTP request. By uploading a malicious file named ../templates/index.html, the attacker commands the operating system to traverse up and out of the safe uploads folder and permanently overwrite the application's core homepage, resulting in complete system defacement.  Project Structure: You are given a few key files:  app.py: The Flask backend server containing the vulnerable /upload route. exploit.py: A simulated attacker's Python script designed to forge a multipart HTTP request and execute the path traversal. templates/index.html: The core HR Portal homepage (The attacker's target). templates/upload.html: The frontend UI for file uploads. Your Task Prove the vulnerability exists by executing the automated exploit script to deface the HR portal. Then, restore the portal and act as the defender. You will patch app.py by implementing an Extension Allowlist, sanitizing the input with secure_filename(), and completely decoupling the file from user input by generating a randomized UUID storage name.  Phase 1: The Attack (System Defacement) Open your terminal and start the vulnerable web server by running: python app.py Open the application in your workspace browser preview. You will see a green, operational Employee HR Portal. Open a second, separate terminal tab in your workspace (leave the server running in the first tab). In the new terminal, launch the automated path traversal attack by running: python exploit.py Go back to your browser preview and refresh the HR Portal homepage (/). Boom! The screen goes black, and you see [ SYSTEM COMPROMISED ]. The attacker successfully escaped the uploads directory and permanently overwrote your index.html template file. Docs: OWASP Path Traversal | Unrestricted File Upload Phase 2: The Defense (The Sandbox) Stop your Flask server in the terminal (press Ctrl+C). CRITICAL STEP: Before you patch the code, you must fix the damage. Open templates/index.html in your code editor. Use your editor's Undo shortcut (Ctrl+Z or Cmd+Z) or RESET the lab to reverse the hacker's defacement and restore the original HR Portal HTML and then Save the file. Open app.py in your code editor and locate the upload_file() function. (Hint: You will need to import secure_filename from werkzeug.utils and the uuid library at the top of your file.) TODO 1 (The Allowlist): At the top of your function, write an if statement to check if the uploaded file has a safe extension (.jpg, .png, .jpeg, .gif). If it doesn't, return a 400 error message. TODO 2 (Sanitization): Pass the file.filename through Werkzeug's secure_filename() utility. This automatically strips out dangerous characters like ../. TODO 3 & 4 (The UUID Sandbox): Never trust user filenames, even sanitized ones. Extract the safe extension, and use uuid.uuid4().hex to generate a completely random string. Combine them to create a brand new, highly secure filename (e.g., 8f4b2c...a91d.png). Update the file.save() function to use this new random name instead of the user's name. Run python app.py to start the secured server. Verify the Block: Go to your second terminal and run python exploit.py again. The attack will fail. If you check your uploads/ folder, you will see the malicious HTML file was safely trapped inside the folder and renamed to a harmless, random UUID.
